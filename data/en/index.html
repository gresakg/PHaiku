Phaiku is a fast and simple but extensible static website manager created with a micro framework Slim. It works without databases and there's no administration section. You can update the site using ftp, sftp or preferably git.

PHaiku is at this stage a proof of concept, meant to be played with, explored and extended. It has a very brief source code, that's ought to be understandable.

The name PHaiku is obviously derived from PHP and haiku, the later being a short poetry form of Japanese origin, consisting of only five verses with respectively 5 - 7 - 5 syllabes. Read more about haiku on Wikipedia.

Features

Multilanguage

Phaiku provides multilanguage features by default. To use them, edit the config file and store each language data in it's own folder.

Themable

Phaiku uses simple php templates for rendering. There's no new templating language to learn, just echo your variables and you're done.

You can have as many themes as you want in your templates folder and change them to your needs. A simple Skela template is just the default you can use or replace.

Widget support

Phaiku provides an easy to use widgets interface. The main and the language menus are implemented as widgets and a simple Haiku widget, that displays random haikus in the sidebar may all serve as examples.

Extendable

You may add your own routes (to the config.php file) and define controllers that controll them. The /contact route serves as example, though is not (yet) fully implemented.

Installation

To install PHaiku, use git to clone it to your public folder, edit configuration file to your needs, and your done.

Alternatively, you can download the zip package and unzip it to your public folder.

Usage

General settings are in the config/config.php file. More specific site settings, language strings, metadata and the like are stored in the _config.php file in your data folder. If you use multilanguage features, there's one _config.php file in each language folder.

To add new pages just add new files with html formatted contend and save them with html extension. You can add directory structure. The directory structure will reflect in the url's.

If you want to access your pages from the menu you must add them to the _menu.php file manually. Just like the config file, the menu file consist of a simple array of key/value pairs.
